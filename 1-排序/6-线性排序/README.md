因为排序算法的时间复杂度是线性的，所以这类算法称之为线性排序。

三种时间复杂度是`O(n)`的排序算法：桶排序、计数排序、基数排序。

## 桶排序

将要排序的数据分到几个有序的桶里，每个桶的数据在单独排序，桶内排序完成以后，每个桶的数据按照顺序依次取出，组成的序列就是有序的。

### 时间复杂度

n个数据均匀分到m个桶中，每个桶内使用快速排序，整个桶的时间复杂度就是`O(n*log(n/m))`，桶的个数接近数据个数 n 时，`log(n/m)`就是一个非常小的常量，时间复杂度接近`O(n)`.

数据的分布很难时均匀的，当所有的数据分配到一个桶内时，时间复杂度为`O(nlogn)`

### 适用场景

数据存储在外部磁盘中，数据量比较大，内存有限，无法将数据全部加载到内存中。

借助桶排序的思想，可以把数据拆分处理。

## 计数排序

**适用于数据所处范围并不大的场景**，把数据分为k个桶，每个桶内存放的数据值都是相同的。

### 核心思想

1. 查找需要排序数组A的数值范围，找出最大值
2. 创建计数数组B，数组的下表对应排序数组A的数值
3. 遍历数组A，数组B记录每个数字的个数
4. 对计数数组B求和，数组存放数据为小于等于此下标的数值数量
5. 创建临时数组C，按照顺序把数字放到数组中的相应位置。
6. 遍历数组A，得到数据a，查找对应下标的数组B中的值b，则a在顺序数组C中的位置为b-1
7. 每遍历一个数字，数组B对应位置上的数字需要减一

### 适用场景

计数排序只能用在数据范围不大的场景，如果数据范围比要排序的数据量还多，就不适合计数排序了。

计数排序只能给非负整数排序，如果是其他数据类型，需要试着转换。

## 基数排序

基数排序对排序的数据有要求的，需要可以分割出独立的“位”进行比较，而且位之间有递进关系，如果a数据的高位比b数据大，那剩下的低位就不用比较懒，除此之外，每一位的数据范围不能太大，耀可以用线性排序徐算法来排序，否则，基数排序的时间复杂度无法做到`O(n)`了
